# 美容院护理排程系统 - 桌面客户端

基于 Electron 构建的美容院智能排程桌面应用，包含护理排期、冲突校验、双向撮合、契合排序四大核心模块。

---

## ✨ 功能特性

### 📅 护理排期
- **美容床建档** — 管理美容床位信息，支持位置、设备备注
- **顾客登记** — 记录顾客基本信息及肤质类型
- **美容师登记** — 管理美容师资料、擅长项目、擅长肤质
- **新建预约** — 选择床位/顾客/美容师/项目/时段，自动推算结束时间
- **时间线视图** — 甘特图式可视化展示各美容床的预约占用情况

### 🛡️ 冲突校验
- **时段重叠检测** — 创建预约时自动校验同一美容床的时段冲突
- **全局冲突扫描** — 一键检测所有日期的预约冲突
- **退订时段释放** — 取消预约后自动释放时段，可被新预约使用
- **占用率统计** — 实时显示各美容床的使用率百分比

### 🤝 双向撮合
- **双向意愿登记** — 支持顾客选美容师、美容师选顾客两种方向
- **互选成交判定** — 只有双方都登记了意愿才撮合成功
- **单方意愿不成交** — 仅有单方意愿的配对归入待撮合状态
- **撮合取消** — 取消撮合后双方意愿恢复为待定

### ⭐ 契合排序
- **肤质档案建立** — 记录肤质类型、敏感度、水润度、弹性度、肌肤关注点
- **契合度评分** — 从肤质匹配、专长匹配、肤况评估、历史评分四个维度计算（满分100分）
- **契合度排行** — 所有顾客-美容师配对按总分降序排列，金银铜牌标识
- **专长智能匹配** — 顾客关注点自动映射到美容师擅长项目

### 💾 数据备份与恢复
- **一键导出** — 将所有数据导出为 JSON 备份文件
- **一键导入** — 从备份文件恢复全部数据
- **安全提示** — 导入前二次确认，防止误覆盖
- **数据持久化** — 使用本地存储，关闭后数据不丢失

## 🚀 快速开始

### 环境要求
- Windows 10 / Windows 11
- Node.js 18 或更高版本（首次启动会自动检测）

### 启动方式

**方式一：双击启动（推荐）**

直接双击 `启动系统.bat` 即可运行。
- 首次运行会自动安装依赖，请耐心等待
- 后续启动直接进入系统

**方式二：命令行启动**

```bash
# 进入项目目录
cd beauty-salon-scheduler

# 安装依赖（首次运行）
npm install

# 启动应用
npm start
```

## 💡 使用指南

### 基础流程

1. **添加基础数据**
   - 点击「美容床建档」添加美容床位
   - 点击「顾客登记」录入顾客信息
   - 点击「美容师登记」录入美容师及其专长

2. **创建预约**
   - 点击「新建预约」，选择床、人、项目、时段
   - 系统自动检测冲突，有冲突会提示

3. **双向撮合**
   - 切换到「双向撮合」模块
   - 点击「登记意愿」分别录入顾客和美容师的意愿
   - 点击「执行撮合」自动配对双向意愿

4. **契合排序**
   - 切换到「契合排序」模块
   - 点击「建立肤质档案」完善顾客肤质信息
   - 点击「计算契合度」生成契合度排行榜

5. **数据备份**
   - 左侧栏底部点击「导出数据」备份全部数据
   - 或通过菜单「文件 → 导出数据备份」操作

## 🔧 打包成独立可执行文件

如需打包成 .exe 单文件，无需安装 Node.js 即可运行：

```bash
# 安装打包工具
npm install electron-builder --save-dev

# 打包
npx electron-builder --win --x64
```

打包后的文件在 `dist/` 目录下。

## 🗂️ 数据存储位置

应用数据存储在系统用户目录下，卸载应用不会丢失数据：

```
Windows: C:\Users\<用户名>\AppData\Roaming\beauty-salon-scheduler\
```

可通过菜单「工具 → 打开数据目录」快速访问。
